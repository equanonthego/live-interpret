# 발언권 이양형 Q&A (텍스트 읽기) 설계

작성일: 2026-07-15
대상: live-interpret (한국어 강의 동시통역 앱)

## 1. 목표

지금은 단방향이다 — 강의자(한국어)가 말하면 청자들이 각자 언어로 듣는다. 청자는 듣기 전용.
여기에 **질의응답**을 더한다: 청자가 질문하고 싶으면 손을 들고, 강의자가 그 청자에게
마이크(발언권)를 넘기면, 청자가 자기 언어로 질문하고 — 강의자는 그 질문을
**컴퓨터 화면에서 한국어 텍스트로 읽는다**(오디오 재생 없음).

예: 한국어 강사 강의 중 베트남 학생이 질문 → 강사가 마이크를 넘김 → 학생이 베트남어로 질문
→ 강사 화면에 그 질문이 한국어 텍스트로 뜸 → 강사가 마이크를 회수해 한국어로 답변 →
답변은 기존 단방향으로 모든 청자에게 각자 언어로 전달(질문한 학생 포함).

## 2. 핵심 개념 — "일방향, 화자만 바뀜"

마이크를 넘기는 것은 **양방향이 아니다.** 어느 순간에도 구조는 "한 명이 말하고 나머지가
각자 언어로 듣는" 일방향이며, **누가 발언권(마이크)을 쥐었느냐에 따라 출발 언어만 바뀐다.**

> 통역 모델 = **발언권을 쥔 현재 화자 → 각 청취자 언어로 통역**

브릿지는 `translationConfig`에 `targetLanguageCode`만 지정하고 **출발 언어는 Gemini가
자동 감지**하므로(현재 코드 그대로), 화자가 강의자(한국어)든 질문자(베트남어)든 같은
브릿지 구조가 방향에 무관하게 처리한다.

## 3. Q&A 흐름

1. 청자가 watch 화면에서 **"질문하기(손들기)"** 탭 → 강의자 broadcast 화면 **대기열**에 표시
2. 강의자가 대기자 중 한 명에게 **"마이크 넘기기"** → 그 청자만 임시로 발행(publish) 허용,
   동시에 강의자 마이크는 잠시 물러남(발언권은 한 번에 한 명)
3. 청자가 자기 언어로 질문 → **[질문자 언어]→ko 브릿지**가 생성되어 질문 오디오를 구독,
   **한국어 텍스트**를 생성해 강의자 화면에 표시(오디오는 재생하지 않음)
4. 질문이 끝나면 강의자가 **"마이크 회수"** → 질문자 발행 권한 회수, [질문자 언어]→ko 브릿지 종료
5. 강의자가 한국어로 답변 → 기존 정방향 브릿지들(ko→각 언어)로 **모든 청자에게 각자 언어** 전달

한 번에 한 명만 발언권을 갖는다(single floor). 손들기 대기열은 순서 파악용.

## 4. 아키텍처

```
[정방향 — 강의]  강의자(ko) ──▶ LiveKit ──▶ ko→en/zh/ja/vi 브릿지들 ──▶ 청자별 언어

[질문]  질문자(vi, 발언권 부여됨) ──▶ LiveKit ──▶ vi→ko 브릿지
                                                    │ (오디오 통역 + outputAudioTranscription)
                                                    ▼
                                    한국어 텍스트만 data 메시지로 발행
                                                    ▼
                                    강의자 broadcast 화면 Transcription 패널에 표시
```

- 질문자의 원음(베트남어)은 어느 누구에게도 재생하지 않는다 — vi→ko 브릿지만 소비.
- 질문 통역 오디오(한국어)도 재생하지 않는다 — 텍스트만 사용. (브릿지는 오디오를 생성하지만
  강의자 화면은 텍스트만 렌더. 비용은 동일.)

## 5. 재사용 (이미 구현되어 있음)

- **통역 텍스트 생성·발행**: `src/lib/translation-bridge.ts`
  - `outputAudioTranscription: {}` (setup) — 통역 오디오의 텍스트를 함께 생성
  - `publishTranscriptionText()` — 그 텍스트를 LiveKit data(topic `"transcription"`)로,
    `attributes.language === targetLanguage`인 참가자에게 발행
- **텍스트 표시 패널**: `src/app/session/[id]/watch/page.tsx`의 transcription 수신·렌더 로직
  → broadcast 페이지로 이식
- **브릿지 클래스**: 방향 무관(출발 언어 자동 감지)이라 그대로 사용

## 6. 새로 추가 (한정적)

### 6.1 발언권 제어 (floor control)
- 세션매니저에 **현재 발언자(floor holder)** 상태와 **손든 청자 대기열** 추가
- 신호 전달은 LiveKit data 메시지(예: topic `"floor"`): `raise-hand`, `grant`, `revoke`, `release`
- 한 번에 한 명 보장(grant 시 이전 발언자 자동 release)

### 6.2 청자 임시 발행 권한
- 현재 청자 토큰은 듣기 전용(`canPublish=false`)
- 발언권을 받은 동안만 발행 허용 — 발언권 부여 시 **새 토큰 재발급**(`canPublish=true`,
  마이크 소스 한정) 또는 `updateParticipant` 권한 갱신. (구현 시 정본 API 확인)

### 6.3 broadcast UI
- **대기열 패널**: 손든 청자 목록 + 각자 "마이크 넘기기" 버튼, 현재 발언자 표시 + "회수"
- **Transcription 패널**: 현재 질문의 한국어 텍스트 실시간 표시(watch 코드 재사용)
- 강의자 참가자에 `language=ko` 속성 설정 → vi→ko 브릿지 텍스트 수신 대상이 됨

### 6.4 watch UI
- **"질문하기(손들기)" 버튼** + 상태 표시(대기 중 / 발언 중 / 종료)
- 발언권 받으면 마이크 캡처·발행 시작(푸시투톡), 회수되면 중지

### 6.5 브릿지 구독 대상 일반화
- 현재 브릿지는 `organizerIdentity`(강의자)를 구독하도록 고정
- **현재 발언자**를 구독하도록 매개변수화 (정방향은 강의자, 질문 브릿지는 질문자)

## 7. 비용

- **정방향(강의)**: 기존과 동일 (활성 언어 수 × 세션 시간, 침묵 포함).
- **질문(역방향)**: [질문자 언어]→ko 브릿지 1개가 **질문하는 시간만큼만** 과금.
  텍스트만 써도 브릿지는 동일하게 돌아가므로 비용은 오디오 재생 여부와 무관.
  한 번에 한 명이라 질문당 수 센트 수준, 상시 부담 없음.

## 8. 에러·엣지 처리

- 발언권 부여했는데 질문자가 말 안 하고 이탈 → 타임아웃 또는 강의자 수동 회수로 브릿지 종료
- 질문자 연결 끊김 → 발언권 자동 release, 브릿지 종료, 대기열에서 제거
- 여러 명이 거의 동시에 손들기 → 대기열에 순서대로, 강의자가 선택
- 발언권 부여 실패(토큰/권한) → 강의자 화면에 오류 표시, 발언권 원복

## 9. 검증 방법

1. 강의자(한국어) 방송 중, 베트남어 청자가 "질문하기" → 강의자 화면 대기열에 뜨는지
2. 강의자가 마이크 넘기기 → 질문자 폰에서 마이크 활성화되는지
3. 질문자가 베트남어로 말하면 → 강의자 화면에 **한국어 텍스트**가 실시간으로 뜨는지
   (오디오는 안 나와야 함)
4. 강의자가 마이크 회수 → 질문 브릿지 종료(로그), 발행 권한 회수 확인
5. 강의자가 한국어로 답변 → 모든 청자(질문자 포함)가 각자 언어로 듣는지
6. 비용: 질문 브릿지가 질문 시간에만 뜨고 종료되는지 로그로 확인

## 10. 범위 밖 (YAGNI)

- 질문을 강의자 외 다른 청자에게도 통역 (강의자만으로 결정)
- 여러 명 동시 발언 / 패널 토론
- 질문 오디오(한국어 음성) 재생 — 텍스트 읽기로 결정
- 질문 텍스트 로그 저장·다시보기
